# Hybrid Customer Churn Prediction with NLP Sentiment Integration

This repository contains an advanced customer churn prediction pipeline that combines traditional tabular customer data (billing, demographics, and usage metrics) with unstructured text feedback. Utilizing **Python**, **spaCy**, and **scikit-learn**, this project demonstrates how natural language processing (NLP) can extract hidden sentiment signals to dramatically improve machine learning classification performance.

---

## 📊 Project Overview & Architecture

Predicting customer churn purely from numeric metrics (like monthly charges or contract type) often leaves blind spots. This project builds a hybrid machine learning model that merges tabular features with tokenized text embeddings, providing a holistic 360-degree view of customer risk.

### Core Architecture Steps:
1. **Tabular Baseline:** Cleaned and encoded traditional user metrics using `pandas` and `scikit-learn`.
2. **NLP Text Preprocessing:** Engineered a custom text-cleaning pipeline with `spaCy` to handle tokenization, punctuation removal, and lemmatization.
3. **Feature Extraction:** Transformed cleaned text feedback into numerical representations using **TF-IDF (Term Frequency-Inverse Document Frequency)** vectorization.
4. **Hybrid Ensemble:** Concatenated tabular and text feature matrices to train a unified **Random Forest Classifier**.

---

## 🔍 The Data Leakage Audit (Engineering Highlight)

During development, the initial hybrid model achieved a perfect **1.00 Precision, Recall, and Accuracy score**. Rather than accepting this at face value, a thorough diagnostic audit was conducted to uncover systemic **Data Leakage**.

By extracting and analyzing the model's **Feature Importances**, it was discovered that the text feedback variables contained highly distinct linguistic artifacts generated by the synthetic data engine. Highly specific structural keywords completely dominated the model's decision-making process:

* **Churn Signifiers:** Keywords like `recently`, `decide`, `ultimately`, and `switch` heavily flagged churning accounts.
* **Retention Signifiers:** Keywords like `satisfied`, `appreciate`, and `reasonable` signaled loyal accounts.

An evaluation of the top 50 features revealed that **45 were NLP text tokens**, while only **5 were traditional tabular features**. This project highlights the critical importance of feature auditing in production machine learning pipelines to prevent models from memorizing artifacts over learning generalized behavior.

## 📈 Model Performance Comparison

### 1. True Tabular Baseline (No Text Features)
The baseline model established a realistic human benchmark, capturing exactly half of the churning customer base.
* **Overall Accuracy:** 79%
* **Class 1 (Churn) Precision:** 0.63
* **Class 1 (Churn) Recall:** 0.50

### 2. Hybrid NLP Model (Tabular + TF-IDF)
Integrating the text features allowed the model to capitalize on the distinct linguistic partitions in customer feedback, pushing performance to near-perfection.
* **Overall Accuracy:** 1.00
* **Class 1 (Churn) Precision:** 1.00
* **Class 1 (Churn) Recall:** 1.00
